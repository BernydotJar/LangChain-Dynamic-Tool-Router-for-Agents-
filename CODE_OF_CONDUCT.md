# Code of Conduct

## Our Standard

This project expects professional, respectful collaboration.

Examples of expected behavior:

- Use welcoming and inclusive language.
- Respect different viewpoints and experience levels.
- Give constructive technical feedback.
- Focus criticism on ideas, code, documentation, and behavior.
- Keep security discussions responsible and evidence-based.

Examples of unacceptable behavior:

- Harassment or personal attacks.
- Discriminatory language.
- Sharing private information without permission.
- Posting secrets, tokens, credentials, or private customer data.
- Misrepresenting the project as production security infrastructure when it is currently developer preview.

## Scope

This code of conduct applies to repository issues, pull requests, discussions, documentation contributions, and other project spaces.

## Enforcement

Project maintainers may remove comments, close issues, reject contributions, or limit participation when behavior does not meet this standard.

## Reporting

Open a GitHub issue or contact the maintainer through the repository profile when reporting conduct concerns. Do not include sensitive private information in public issues.
