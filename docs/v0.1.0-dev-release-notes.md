# v0.1.0-dev Release Notes

Release status: developer preview

## Product Positioning

```text
Runtime Tool Authorization
for AI Agents

Never expose every tool.
Expose the right tool.
```

Dynamic Tool Router provides a local developer-preview implementation of request-time tool authorization for AI agents.

## What Works

- Runtime policy evaluation.
- Authorized tool injection.
- Fallback routing for unavailable tools.
- User, tenant, plan, role, permission, context, and MCP-server policy dimensions.
- File-backed JSON policy loading.
- JSONL audit persistence.
- JSON audit export.
- LangChain-style adapter shape.
- LangGraph-style middleware shape.
- Dependency-gated LangChain/LangGraph integration tests.
- Static dashboard documentation and sample direction.

## Install

```sh
python -m pip install -e .
```

## Verify

```sh
python -m json.tool feature_list.json
PYTHONPATH=src python -m unittest discover -s tests
python examples/basic_agent/run_example.py
PYTHONPATH=src python -m unittest discover -s tests/integration
```

## Package Build

```sh
python -m pip install -e .[dev]
python -m build
```

## Known Limitations

- Developer preview only.
- No hosted policy API.
- No production auth-provider integration.
- No database-backed policy store.
- No tamper-proof audit sink.
- No authenticated admin dashboard.
- No compliance certification.
- Optional LangChain/LangGraph tests may skip if dependencies are absent.
