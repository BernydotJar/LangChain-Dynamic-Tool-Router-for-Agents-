# Admin Dashboard

The repository includes a static admin dashboard example for developer-preview visibility.

## Purpose

The dashboard is intended to help reviewers understand:

- which tools were allowed,
- which tools were not allowed by policy,
- which fallback behavior was applied,
- what audit events were produced,
- how policy decisions might appear in a future admin product.

## Developer-Preview Boundary

The dashboard is static and unauthenticated.

It is not:

- a production admin console,
- an authenticated policy editor,
- a hosted dashboard,
- a compliance report,
- a tamper-proof audit viewer.

## Data Source

Use exported audit JSON or sample audit output generated by the basic demo.

Run:

```sh
python examples/basic_agent/run_example.py
```

Then inspect the printed `Audit export:` path.

## Future Direction

A production dashboard should add:

- login and role-based admin access,
- policy editing workflow,
- policy approval gates,
- audit filtering,
- tenant-level views,
- immutable audit backing store,
- export controls,
- integration with observability or SIEM tooling.
