## Product Identity

```text
Runtime Tool Authorization
for AI Agents

Never expose every tool.

```

## Product Category

Infrastructure for AI agents.

## Core Thesis

Everyone is building AI agents. Few teams are building the infrastructure layer that decides which tools an agent is actually allowed to use at runtime.

Dynamic Tool Router is the authorization layer between:

```text
LLMs -> Agents -> Tools -> Your Infrastructure
```

## Release Bar

The developer preview is not ready until a stranger can:

1. understand the product in 30 seconds,
2. install it locally,
3. run a demo,
4. see allowed and unavailable tools,
5. inspect persisted audit output,
6. understand the security boundary,
7. see a credible roadmap,
8. know how to evaluate it as a design partner.

## Harness Rules

- No giant unreviewable feature dumps.
- Every SHIP feature needs requirements, design, tasks, approval, implementation, verification, review, and closure.
- Commercial urgency does not override lifecycle evidence.
- Product claims must not exceed verified behavior.
